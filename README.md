# 📝 Blogicum

Blogicum — учебная блог-платформа, разработанная на Django.

Пользователи могут публиковать записи, прикреплять изображения, комментировать публикации, редактировать профиль и управлять собственным контентом.

---

# 🎯 Цель проекта

Основная задача заключалась в работе с:

* Class-Based Views (CBV);
* системой пользователей Django;
* авторизацией и регистрацией;
* формами и обработкой данных;
* правами доступа;
* пагинацией;
* загрузкой изображений;
* комментариями;
* оптимизацией запросов;
* переиспользованием логики через миксины.

---

# ✅ Что реализовано

### Пользователи

* регистрация пользователей;
* авторизация и выход из системы;
* редактирование профиля;
* изменение пароля;
* страницы пользователей.

### Публикации

* создание публикаций;
* редактирование публикаций;
* удаление публикаций;
* отложенные публикации;
* прикрепление изображений;
* отображение публикаций по категориям.

### Комментарии

* создание комментариев;
* редактирование комментариев;
* удаление комментариев;
* подсчёт количества комментариев для каждой публикации.

### Дополнительно

* пагинация по 10 публикаций на страницу;
* кастомные страницы ошибок 403, 404 и 500;
* административная панель Django;
* файловый backend для отправки писем;
* оптимизация запросов через `select_related()` и `annotate()`.

# ⚙️ Установка проекта

### Клонирование репозитория

```bash
git clone https://github.com/VirentLilium/django-sprint4.git
cd django-sprint4
```

### Создание виртуального окружения

```bash
python -m venv venv
```

### Активация виртуального окружения

Windows:

```bash
venv\Scripts\activate
```

Linux / macOS:

```bash
source venv/bin/activate
```

### Обновление менеджера пакетов

```bash
python -m pip install --upgrade pip
```

### Установка зависимостей

```bash
pip install -r requirements.txt
```

# 🚀 Запуск проекта

Перейдите в директорию Django-проекта:

```bash
cd blogicum
```

Примените миграции:

```bash
python manage.py migrate
```

Создайте суперпользователя:

```bash
python manage.py createsuperuser
```

Запустите сервер разработки:

```bash
python manage.py runserver
```

После запуска проект будет доступен по адресу:

```text
http://127.0.0.1:8000/
```
